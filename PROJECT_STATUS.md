# Project Status Ledger (Mihon / Neo Manga v3)

This document serves as the persistent, living ledger for the Neo Manga v3 project. It tracks the historical log of architectural modifications, our current capabilities, and our future roadmap. **This file will be updated iteratively as new features, modifications, and fixes are deployed.**

## [What Happened]

### Project Overview
Neo Manga v3 began as a fork/evolution of the Mihon open-source manga reader. The goal was to pivot the standard, multi-source global reader into a highly curated, premium, Arabic-localized application utilizing strict design tokens (Phase 5/6 aesthetics).

### Current vs. Original
- **Original State:** Mihon relied on a highly decentralized structure with a "Browse" tab where users manually managed extensions, sources, and global updates. The UI was functionally dense but visually standard.
- **Current State:** The application architecture has been radically restructured into a "Zero-Loading, Local-First" design. The legacy multi-source global feed has been replaced entirely with a premium **Arabic Home Dashboard** (`DashboardScreen`) that serves as the immediate entry point. Design aesthetics strictly follow "Phase 5/6 Tokens": pitch-black backgrounds (`0xFF000000`) and cyber-teal accents (`0xFF00E5FF` at `0.15f` alpha). The data pipeline has been locked to a unified, pre-fetched priority cache ("Team X"), shifting away from asynchronous multi-source polling to instantaneous local-first hydration.

### Bugs & Fixes
- **Deduplication Page Calculation Loop Fix:** Resolved the 15-item trap where deduplication filtering reduced the initial cache size below 20 items. Replaced the dynamic size-derived page calculation formula with an independent, state-controlled `nextPageToFetch` pagination state counter, decoupling page tracking from `masterCache` size fluctuations.
- **Recommendations 20-Item Lock & Hydration Deadlock:** Resolved the 20-item viewport lock by gating pagination triggers behind `isInitialHydrationComplete` and dynamically deriving the next page to fetch from loaded cache size to prevent background prefetch page-skipping race conditions.
- **Recommendations Blank Screen Startup & Source Fallback Bug:** Resolved a runtime state hydration bug where the recommendations screen loaded blank if the priority catalog source ("Team X") was unhydrated or uninstalled. Injected diagnostic logs across `init`, fallback fetches, cache additions, and state emitters. Added a safe fallback to the first available CatalogueSource if Team X is null to ensure dense recommendations delivery.
- **Recommendations Page Counter Drift & Hard Wall Jitter:** Fixed a page counter drift bug where scrolling up and down repeatedly triggered speculative fetches for higher out-of-bounds pages, causing premature page exhaustion. Also decoupled the viewport shifting down from the network prefetch coroutine to eliminate the scroll-blocking "hard wall" artifact.
- **Recommendations Screen Blank Render (DI Scope Mismatch):** The recommendations screen loaded blank because `GetUnifiedGlobalCatalogUseCase` was registered as a standard `factory` in Injekt, producing a fresh instance with empty cache state inside `RecommendationsScreenModel`. Changed to a `singleton` via `addSingletonFactory` inside `DomainModule.kt` to share the pre-warmed cache pool correctly.
- **Updates Tab InjektionException Crash:** The `UpdatesScreenModel` was requesting the Android `Context` dependency directly from Injekt, which failed at runtime with an `InjektionException`. Resolved by modifying `updateLibrary()` signature to accept context explicitly, passing `LocalContext.current` from the Compose layer (`UpdatesTab.kt`).
- **InjektionException Crash:** The `GetUnifiedGlobalCatalogUseCase` was missing from the DI graph. Fixed by wiring it into the dependency registry (Injekt).
- **Infinite Loading Loops / UI Blocking:** The initial `DashboardScreenModel` used `awaitAll()` and `firstOrNull()` calls inside the init block, blocking the UI thread and hiding the screen behind an infinite spinner. Fixed by enforcing a "Zero-Loading" architecture with fire-and-forget background hydration and immediate default states.
- **Navigation Scope Crashes:** Attempting to call Voyager navigation from non-composable lambda scopes. Fixed by passing the `TabNavigator` explicitly down the composable tree or routing via click callbacks.
- **Carousel Missing on Cold Start:** The Featured Carousel disappeared entirely if the memory cache was empty on cold start because `firstOrNull()` stopped collecting the flow and the UI returned early. Fixed by switching `DashboardScreenModel` to `collectLatest` for reactive updates, and adding a premium cyber-teal shimmer placeholder in `DashboardScreen.kt` while the background fetch populates.
- **Carousel Stuck in Loading (Pipeline Deadlock):** The UI stayed stuck in a shimmer spinner if the cache was permanently empty. Refactored `GetUnifiedGlobalCatalogUseCase.kt` to immediately fallback to querying the direct "Team X" network extension source (`getPopularManga`) inside the flow when the cache misses. The data is now reactively returned and instantly cached.
- **Invisible Carousel / UI Collapse:** The `HorizontalPager` was completely collapsing and rendering invisible because it was measured inside a `Column` with `verticalScroll()` (unbounded constraints). Fixed by migrating the root container of `DashboardScreen.kt` to a `LazyColumn` and wrapping all UI sections in `item {}` blocks.
- **Missing Cover Images:** The carousel images were completely invisible because Coil 3's `AsyncImage` was receiving a raw `Manga` object. Fixed by switching to an explicit `ImageRequest.Builder` targeting `thumbnailUrl`.
- **Carousel Blank Rendering & Hydration Race Conditions:** The Featured Carousel rendered blank or stayed stuck in a loading shimmer on startup due to eager cold-flow execution of `await()`, race conditions on source loading in `first()`, and RTL page mirroring in Compose Pager. Resolved by converting `await()` to a standard flow builder, adding a 5-second timeout and 3x retry lookup loop for the Team X extension, preloading the catalog in `DashboardScreenModel`, using `.collect {}` to prevent implicit cancellations, disabling RTL mirroring on `HorizontalPager` with `reverseLayout = false`, declaring `PageSize.Fill`, and adding a `ColorPainter` fallback to `AsyncImage` to ensure the layout remains visible even if the thumbnail is null.
